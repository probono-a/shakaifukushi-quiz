# 非エンジニア向けアクセシビリティ改善計画

## 目標

**公開リポジトリの README を読めば使い始められる**状態にする。

- ターゲット: AI チャット（Gemini, ChatGPT など）を日常的に使える人
- Python / ターミナルの知識なしで問題追加・クイズ実施が完結する（初回セットアップは除く）

---

## 想定ワークフロー（完成後）

```
[初回のみ] GitHub から ZIP ダウンロード → setup.bat → run.bat
                                            （README の手順どおり）
     ↓
[毎回] run.bat ダブルクリック → ブラウザが開く
     ↓
  ┌─────────────────────────────────────────┐
  │  ダッシュボード ／ 問題入力 ／ クイズ  │
  │  どの画面からでも他の画面へ 1 クリック  │
  └─────────────────────────────────────────┘
     ↓
[問題追加] 過去問を見ながらフォームに手打ち
           → Gemini に解説を書いてもらいコピペ → 保存
     ↓
[復習]     クイズ画面で解く
```

---

## 変更内容

### Task 1: `POST /api/questions` エンドポイントの追加

**ファイル**: `routers/questions.py`

現状 PUT（編集）はあるが POST（新規作成）がない。

追加する処理：
- リクエスト body を受け取り、`questions` テーブルに INSERT
- ID は `{edition}_{question_number}` 形式で自動生成（衝突時は 422 エラーを返す）
- `is_multiple` は `correct_options` の数から自動判定
- JSON フィールド（options, correct_options, keywords, reference_links）は自動シリアライズ
- 成功時に `{"ok": true, "id": "<生成したID>"}` を返す

```json
// リクエスト body のイメージ
{
  "edition": 38,
  "curriculum": "新",
  "subject": "社会福祉の原理と政策",
  "question_number": 1,
  "question_type": "通常",
  "case_text": "",
  "question_text": "問題文...",
  "options": ["選択肢1", "選択肢2", "選択肢3", "選択肢4", "選択肢5"],
  "correct_options": [3],
  "explanation": "解説...",
  "keywords": ["キーワード1"],
  "reference_links": []
}
```

### Task 2: `static/editor.html` の作成

ブラウザだけで問題を登録できる単ページフォーム。

**UI 構成**:

```
┌──────────────────────────────────────────────────┐
│ [ダッシュボード] [問題入力] [クイズ]  ← 共通ナビ │
├──────────────────────────────────────────────────┤
│ 問題入力                                         │
├──────────────────────────────────────────────────┤
│ 回次（例: 38）    [____]                         │
│ カリキュラム      [新 ▼]                         │
│ 科目             [社会福祉の原理と政策 ▼]        │ ← GET /api/subjects
│ 問題番号         [____]                          │
│ 問題タイプ       [通常 ▼]                        │
├──────────────────────────────────────────────────┤
│ 事例文（任意）                                   │
│ [                                              ] │
├──────────────────────────────────────────────────┤
│ 問題文 *                                         │
│ [                                              ] │
├──────────────────────────────────────────────────┤
│ 選択肢（正答肢のボタンをクリックしてチェック）   │
│ ☐ 1. [______________________________]           │
│ ☐ 2. [______________________________]           │
│ ☐ 3. [______________________________]           │
│ ☐ 4. [______________________________]           │
│ ☐ 5. [______________________________]           │
├──────────────────────────────────────────────────┤
│ 解説（Markdown 可） *                            │
│ [                                              ] │
├──────────────────────────────────────────────────┤
│ キーワード（Enter で追加）                       │
│ [____________] [追加]   タグ1 ×  タグ2 ×        │
├──────────────────────────────────────────────────┤
│               [保存する]                         │
└──────────────────────────────────────────────────┘
```

**実装方針**:
- 既存の `style.css` を流用してデザインを統一
- 科目プルダウンは `GET /api/subjects` から動的取得
- 正答肢は選択肢左のボタンでトグル（複数選択対応）
- 保存成功後にフォームをリセット＋「保存しました」トースト表示
- 保存失敗時（ID 重複など）はエラーメッセージを表示

### Task 4: README の書き直し

**ファイル**: `README.md`（上書き）

公開リポジトリを見た非エンジニアが README だけで使い始められるように書き直す。

**構成**:

```

（1 行キャッチコピー）

## これは何？
（3 行以内で概要）

## 使い方
### ステップ 1: Python と uv をインストールする
### ステップ 2: アプリを入手する（ZIP ダウンロード）
### ステップ 3: セットアップして起動する（setup.bat → run.bat）
### ステップ 4: 問題を入力する（/editor）
### ステップ 5: クイズで復習する（/quiz.html）

## 問題の入力方法（詳細）
- 過去問を見ながらフォームに入力
- Gemini などに解説を書いてもらいコピペする使い方を説明

## よくある質問
- 「サーバーが起動しない」
- 「問題が表示されない（DB が空）」

## 開発者向け（既存の詳細セクション）

- API エンドポイント一覧
- プロジェクト構成
```

**方針**:
- 現在の README は開発者向け情報が先に来ている → 使い方を最初に持ってくる
- `localhost:8000` という URL だけでなく「ブラウザのアドレスバーに入力する」という説明を補う
- スクリーンショットは現状のもの（`docs/screenshots/`）を流用
