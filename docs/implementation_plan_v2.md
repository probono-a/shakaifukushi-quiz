# 社会福祉士過去問アプリ 実装計画 v2.0

## 概要
OCR による画像解析から、公式サイトの HTML およびテキスト PDF を活用した「ハイフィデリティ（高精度）抽出」へと戦略をシフトします。これにより、36回以降の試験データについては 100% のテキスト精度を担保します。

## フェーズ 1: 基礎環境の再整備 (完了)
- ディレクトリ構成の定義
- 必須ライブラリ（BeautifulSoup4, PyMuPDF）の導入確認

## フェーズ 2: 高精度抽出パイプラインの構築
1. **データ収集ツールの実装 (`tools/fetch_exam.py`)**
   - 社会福祉振興・試験センターの過去問ページから HTML と PDF を自動取得
   - `data/source/{edition}th/` への整理保存
2. **正答抽出ロジックの実装**
   - 正答 PDF（テキスト型）から問題番号と正解のペアを抽出
   - `answer_key_{edition}th.json` の生成
3. **HTML パーサーの実装 (`tools/parse_html_exam.py`)**
   - 音声読み上げ用 HTML を BeautifulSoup4 で解析
   - 科目、問題文、事例、選択肢を構造化
   - 正解データとの紐付け
4. **統合 JSON の生成とバリデーション**
   - `exam.json` の生成
   - 表記ゆれ（全角・半角スペース）の自動修正

## フェーズ 3: データベース統合
1. **インポーターの更新**
   - 生成された JSON を SQLite に一括インポート
   - `is_reviewed` フラグの自動付与（HTML 由来は信頼度が高いため）

## フェーズ 4: アプリケーション実装 (継続)
- FastAPI バックエンドの構築
- モダンな UI による解答画面の実装
- 学習統計ダッシュボード

## 変更されたツールの方針
- `pdf_to_png.py`: レガシー（35回以前）用、および図表の切り出し用として維持。
- `parse_html_exam.py`: 新規作成。今後の主流。
- `fetch_exam.py`: 新規作成。手動ダウンロードを廃止。
