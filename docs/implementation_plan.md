# 社会福祉士過去問アプリ 実装計画

要件定義に基づき、アプリ開発を以下の6つのフェーズに分けて進める計画です。

## フェーズ 1: 基礎リポジトリと環境構築

アプリの土台となるディレクトリ構成と必須ファイルを作成します。

* **タスク**
    * リポジトリのディレクトリ構成の作成（`converter`, `prompts`, `static`, `data/pdf`, `data/images` など）
    * `requirements.txt` の作成（FastAPI, Uvicorn, requests, BeautifulSoup4 など）
    * Windows 用スクリプトの作成（`setup.bat`, `run.bat`）
    * FastAPI のエントリーポイント（`main.py`）のモック作成

## フェーズ 2: データ準備ツールの実装

過去問データの収集と DB への取り込みフローを構築します。

* **タスク**
    * **PDF ダウンローダー (`converter/download_pdfs.py`) の実装**:
        * 日本ソーシャルワーク教育学校連盟のサイトから問題・正答 PDF を自動ダウンロードする処理
    * **AI 抽出プロンプト (`prompts/pdf_to_json.md`) の作成**:
        * JSON スキーマの定義（`image_paths`, `keywords`, `links` などの構造定義）と、抽出ルールの言語化
    * **GUI エディタ (`tools/quiz_editor.html`) の作成**:
        * AI が抽出した JSON を人間が目視で確認・修正し、`is_reviewed` フラグを管理するためのツール
    * **バリデーションスクリプト (`converter/validate_quiz_json.py`) の実装**:
        * JSON の構造チェック、および全角・半角の間のスペース等の表記ルールを自動チェックする処理
    * **JSON インポーター (`converter/import_json.py`) の実装**:
        * SQLite のテーブル作成ロジック（`questions`, `history`, `subject_mapping`）
        * 修正・検証済み（`is_reviewed: true`）の JSON ファイルを読み込んで DB に INSERT する処理
    * **テスト**: 実際に数問分の JSON を AI で作成し、DB に取り込めるか確認

## フェーズ 3: バックエンド API の実装

フロントエンドと連携するための REST API を実装します。

* **タスク**
    * **出題 API (`GET /api/questions`)**:
        * 科目、カリキュラム、復習モード（間違えた問題のみ等）のフィルタリング機能
        * 旧カリキュラムの MIX/除外ロジックの実装
    * **解答記録 API (`POST /api/history`)**:
        * ユーザーの解答結果（正誤）をDBに保存する処理
    * **ダッシュボード API (`GET /api/stats`)**:
        * 総正答率、科目別正答率などの集計データを返す処理
    * **科目マッピングデータの初期化**: アプリ起動時に新旧科目のマッピングデータを DB にシードする処理

## フェーズ 4: フロントエンド実装（出題画面）

ユーザーが実際に問題を解くための UI（`index.html`）を構築します。

* **タスク**
    * **モダンな UI デザインの適用**: CSS フレームワークやカスタム CSS を用いたリッチな見た目の構築
    * **スタート画面**: 出題条件（科目、カリキュラム、復習モード）の選択 UI
    * **問題解答画面**:
        * 問題文、選択肢、図表（画像）の表示
        * 1択 / 2択問題の判定ロジック
    * **結果・解説表示**:
        * 解答直後に正解とキーワード、参考リンクを表示する UI

## フェーズ 5: フロントエンド実装（ダッシュボード）

学習履歴を可視化する UI（`dashboard.html`）を構築します。

* **タスク**
    * 全体の正答率表示
    * 科目ごとの正答率のグラフ表示（Chart.js などの軽量ライブラリの使用を検討）
    * 苦手科目（正答率が低い科目）のハイライト表示機能

## フェーズ 6: 仕上げ・公開準備

OSS として公開するためのドキュメント整備と最終確認を行います。

* **タスク**
    * 一連のフロー（セットアップ → ダウンロード → JSON作成 → アプリ起動 → 問題解答）の通しテスト
    * `README.md` の執筆（セットアップ手順やデータ作成フローの丁寧な解説）
    * 不要なファイルやハードコードされた値の整理

## 共通ルール（コーディングおよびドキュメント）

開発を進めるにあたり、以下のルールをすべてのファイル（ドキュメント・コード）で徹底します。

* 全角文字と半角文字の間には必ず半角スペースを入れる
* 箇条書きの文末には句点を付けない
* コード内のコメントは過剰にならない程度に、それぞれ何を行っているかわかるように記述する
