## 概要

社会福祉士国家試験の過去問を使ったローカル動作の問題集アプリ。Python 環境があれば PC 上で完結する。GitHub で公開し、使いたい人は Clone して使う。

## コンセプト

* **ローカルファースト**: インターネット接続不要、Firebase 等のクラウドサービス不使用
* **オープンソース**: GitHub で公開、Clone して自分の環境で動かす
* **データ分離**: 著作権への配慮から過去問データはリポジトリに含めず、セットアップ後に自分で変換する
* **PC 特化**: スマートフォン対応はせず、ブラウザで [localhost](http://localhost) を開いて使う

## 技術スタック

* **バックエンド**: Python + FastAPI + Uvicorn
* **データベース**: SQLite（Python 標準ライブラリ、インストール不要）
* **フロントエンド**: HTML / CSS / JavaScript（PC ブラウザ前提でリッチな UI を目指す）
* **問題データ作成**: エディタ上の AI エージェント（Copilot / Antigravity 等）にプロンプトを渡し、PDF から JSON を生成させる手法を採用（プログラムによる自動解析は行わない）
* **開発**: AI エージェントに実装を依頼

## セットアップフロー（README に記載する内容）

### Windows の場合（自動化）

1. git clone [https://github.com/xxx/shakaifukushi-quiz](https://github.com/xxx/shakaifukushi-quiz)
2. フォルダを開き、`setup.bat` をダブルクリック（初回のみ：仮想環境作成＆ライブラリインストール）
3. `python converter/download_pdfs.py` を実行して過去問PDFをダウンロード（`data/pdf/` に保存される）
4. AIエージェントを使ってダウンロードしたPDFからJSONデータを作成し、`data/` 以下に配置
5. `python converter/import_json.py` を実行して JSON を DB に登録
6. `run.bat` をダブルクリックしてアプリを起動
7. ブラウザで [http://localhost:8000](http://localhost:8000) を開く

※ Python, Git がインストールされている前提。環境構築のハードルを下げるため、bat ファイルによる自動化を提供する。

## 問題データ

### カリキュラムについて

第 37 回（2025 年 2 月）から新カリキュラムに移行。科目名・構成が変わっているため新旧を区別して管理する。

| 区分 | 対象回 | 備考 |
| --- | --- | --- |
| 新カリキュラム | 第 37 回・第 38 回 | 現行の出題基準 |
| 旧カリキュラム | 第 36 回以前 | 科目名が異なる |

**新カリキュラム（第 37・38 回）と旧カリキュラム（第 36 回以前）の両方を対象とする。**

出題時のデフォルト動作としては「旧カリキュラムの該当科目も含めて出題（新旧 MIX）」とし、UI 上で「旧カリキュラムを除外する」オプション機能を提供する。

| 旧カリキュラム | 新カリキュラム |

| 相談援助の基盤と専門職 | ソーシャルワークの基盤と専門職 |
| 相談援助の理論と方法 | ソーシャルワークの理論と方法 |

マッピングは SQLite の別テーブルとして持たせ、科目フィルター UI で新旧混在でも正しく分類できるようにする。

### ライセンスに関する注意

* 試験センターの著作権表記あり。問題文の再配布は原則グレー
* **過去問データ（PDF, JSON, DB）はリポジトリに含めない**（.gitignore で除外）
* ユーザー自身がスクリプトで PDF をダウンロードし、AI エージェントに JSON 化させてからアプリに取り込む設計にする
* README にその旨（スクリプト取得と AI エージェントを用いたデータ作成フロー）を明記する

### DB スキーマ（SQLite）

```sql
-- 科目マッピングテーブル
CREATE TABLE subject_mapping (
  subject_old   TEXT,
  subject_new   TEXT,
  subject_group INTEGER
);

-- 問題テーブル
CREATE TABLE questions (
  id               TEXT PRIMARY KEY,  -- 例: "38_85"
  edition          INTEGER,           -- 第何回
  subject          TEXT,              -- 科目名
  question_number  INTEGER,           -- 問題番号
  question_type    TEXT,              -- "通常", "事例" など
  case_text        TEXT,              -- 事例文
  question_text    TEXT,              -- 問題文
  is_multiple      INTEGER,           -- 複数選択フラグ (0 or 1)
  options          TEXT,              -- JSON 配列 (選択肢)
  correct_options  TEXT,              -- JSON 配列 (正解番号 1-5)
  explanation      TEXT,              -- 解説
  keywords         TEXT,              -- JSON 配列
  reference_links  TEXT,              -- JSON 配列
  image_paths      TEXT,              -- JSON 配列
  curriculum       TEXT               -- "new" or "old"
);

-- セッションテーブル（1 回の学習セッションを管理）
CREATE TABLE sessions (
  id          TEXT PRIMARY KEY,  -- UUID
  started_at  DATETIME,
  ended_at    DATETIME,          -- NULL = セッション進行中
  mode        TEXT,              -- "subject" / "wrong_only" / "random" / "edition"
  config      TEXT               -- JSON: 科目リスト・問題数・エディションなどのセッション設定
);

-- 学習履歴テーブル
CREATE TABLE history (
  id           INTEGER PRIMARY KEY AUTOINCREMENT,
  session_id   TEXT,             -- sessions.id への参照
  question_id  TEXT,
  answered_at  DATETIME,
  is_correct   INTEGER,          -- 0 or 1
  subject      TEXT,
  curriculum   TEXT,             -- "new" or "old"
  edition      INTEGER           -- 第何回（questions と JOIN せずに集計できるよう非正規化）
);
```

### 問題データの作成・精度向上フロー (Human-in-the-Loop)

PDF をプログラムで完全にパースして正解データと突合させるのは不確実性が高いため、AI エージェントによる一次抽出と、人間による最終確認・修正を組み合わせた「Human-in-the-Loop」フローを採用し、データの正確性を担保する。

1. **データ準備**: `python converter/download_pdfs.py` を実行し、日本ソーシャルワーク教育学校連盟 ([https://jaswe.jp/kokushiinfo.html](https://jaswe.jp/kokushiinfo.html)) から「問題 PDF」と「正答 PDF」を `data/pdf/` に自動ダウンロードする
  
2. **AI 抽出・一次データ作成**: リポジトリに用意した `prompts/pdf_to_json.md` （抽出指示書）と取得した PDF を AI エージェントに読み込ませる。問題文のパースと正解データの突合を AI に任せ、所定の JSON フォーマットで出力させる
  
3. **人間による確認・修正 (Quality Control)**:
  

* **GUI エディタの活用**: 生成された JSON を `tools/quiz_editor.html` で開き、人間が目視で確認・修正を行う。
* **バリデーション**: `converter/validate_quiz_json.py` を実行し、データ構造や表記ゆれ（全角半角の間のスペース等）に不備がないか機械的にチェックする。
* **修正内容**: 問題文の誤字、選択肢の不備、画像の紐付けなどを GUI 上で修正し、最終的に `is_reviewed` フラグを立てる。

4. **図表の対応**: 問題に図表が含まれる場合、手動でスクリーンショットを取得し `data/images/` に保存する。エディタ上でそのファイル名を `image_paths` に記載する
5. **DB インポート**: 修正・確認が完了した JSON を `data/` に置き、`python converter/import_json.py` を実行して SQLite に登録する

## 機能要件

### 問題出題

* セッション開始時に科目・回（カリキュラム）を選択できる（複数選択可）
* **復習機能**: 「前回間違えた問題のみ」「未解答の問題のみ」といった条件フィルターで出題できる
* 「全科目ランダム」も選択肢として用意
* 出題対象にはデフォルトで旧カリキュラムの類似科目も含める（除外オプションあり）
* 選択形式に応じて UI を切り替える（1 択 / 2 択）

### 解答・解説

* 選択肢を選ぶと即時に正誤を表示
* 2 択問題は 2 つ選んだ時点で判定
* 解答後に以下を表示:
  * 正解番号
    * 問題に紐づくキーワード
    * キーワードに対応する参考リンク
* AI による解説生成は行わない（Gemini in Chrome などで別途調べる想定）

### セッション管理

* セッション開始時に以下のモードを選択できる:
  * **科目指定**: 選択した科目の問題のみ出題（複数科目選択可）
  * **間違えた問題のみ**: 過去に 1 度でも不正解だった問題を対象にセッション開始
  * **ランダム N 問**: 選択した科目・エディションの範囲からランダムに任意の問題数（例: 10 問・50 問）を出題
  * **エディション指定（模擬受験）**: 特定の回（例: 第 38 回）の全問を通しで出題。複数年分のデータが揃った段階で「去年の試験を通しでやってみる」という使い方を想定
* セッションの設定（モード・科目・問題数）と開始・終了日時を `sessions` テーブルに記録する
* **今日のおすすめセッション**: 解答履歴をもとに、バックエンドがルールベースで学習対象を推薦する
  * 推薦ロジック: 科目別正答率が低い順に並べ、最後に学習した日が古い科目を優先し、上位 2〜3 科目を対象としたセッション設定を自動生成する
  * ダッシュボードのトップに「今日のおすすめ」カードとして表示し、ワンクリックでそのセッションを開始できる
  * 学習履歴が蓄積されていない初回は表示しない（一定数の解答履歴がある場合のみ表示）

### 学習記録・可視化（ダッシュボード）

* 解答履歴をセッション単位で SQLite に保存する
* 可視化:
  * **総正答率**: 全解答の正誤を集計した全体数値
  * **科目別正答率（棒グラフ）**: 各科目の正答率を比較。苦手科目を色で強調表示
  * **時系列正答率推移（折れ線グラフ）**: 日次または週次の正答率の変化を表示し、成長過程を可視化
  * **苦手問題リスト**: 問題単位の通算正答率が低い問題 TOP N を表示（`history` を `question_id` で集計）
  * **直近のセッション履歴**: 最近のセッションごとの正答率と問題数を一覧表示
  * **カリキュラム別正答率**: 新カリキュラム（第 37 回以降）と旧カリキュラムの正答率を比較
  * **エディション別正答率**（後回し可）: 回ごとの正答率。問題が揃った段階で「模擬受験モード」として活用

### 管理・運用

* 過去問のインポートは `import_json.py` スクリプト経由で実施する
* 完全にローカルで動作するため、ログインなどの認証機能は不要
* **学習履歴の CSV エクスポート**: ダッシュボードのボタンから解答履歴を CSV 形式でダウンロードできる
  * 出力項目: 問題 ID、科目、エディション（回）、カリキュラム、正誤、解答日時
  * Gemini in Chrome などの AI ツールに貼り付けて学習傾向の分析に活用することを想定
  * バックエンド: `GET /api/history/export` エンドポイントで CSV を返す
